Day-10

clipper模块

说明

感觉B样条那里涉及到太多的矩阵什么的容易晕，先尝试看其他模块的代码会不会有这个问题。

分析

cfg两行

第一个要了解的是lambda这里的使用逻辑，其次是python中的类对象可以不先声明直接调用(调用的时候一同生成这个变量)。

ensure_list函数

这里是将用户传入的数据做一个形式上的统一，比如一个点就是一个坐标，一个线条就是多个坐标，多个线条就是多个一个线条。如果这个时候去for-in遍历得到的元素是不同的，会有问题。所以这里的统一形式是很有必要的。

polynode_contours函数

  • 这里的参数比较特别，是pyclipper.PyPolyNode这个玩意的类型对象。为什么会有这个东西，假如一个图形是正方形减去一个圆，那Contour属性就是第一层，Childs就是里面圆的这一层。如果图形特别复杂，可能里面的形状是由很多图形拼接所形成。

  • 这里的函数就是起记录信息的作用，for c in n.Childs这边还用到了DFS算法调用本函数polynode_contours，将每一个第一层深挖。然后这里还有个性质，就是python函数里对列表这些的修改是能够传回去，所以这里没有用return返回值。

conv_from函数

  • if type(S)==clip.PyPolyNode便是在判断S是否是PyPolyNode对象类型，是的话就创建一个空列表contours用来进行上面一个函数的信息记录。

  • return [np.array([np.array(p)/cfg.scale for p in P]) for P in contours]这里是一个列表推导式，同时也用到np数组对数据一次性处理的特性。还是以图形切割为例，contours记录了所有图形，P是某一个图形，p是某一个图形的某一个点。

  • 同时这里因为某些原因，数据必须是整数，在计算前会全部给乘某个倍数，这里除法就是在消去这个东西。

conv_to函数

  • 上面提到数据必须是整数，这里就是在倍乘，然后再转换成整型数据类型。

  • return tuple([tuple([(int...)])这里为什么要用元组，因为列表的数据是可变，而这个特殊的什么东西对这个有要求。这里也涉及到三个元组的嵌套，最内层将点xy围在一起，其次是将一个完整图形的所有点包在一起，最后打包出去，逻辑顺一下。

offset函数

  • 整体作用是接收一个形状(点集)，通过平移变化大小，这个过程的线线交点也被处理了(平角、尖角、圆角)。
  • S代表处理的原始几何形状；amt代表偏移的距离；join_type为连接处的类型(round圆弧，square切平，miter尖角)；end_type为断口处的类型(closed_polygon为闭合，open…等是其他的样式)；miter只有在join_type=miter的时候才起作用，处理一定东西。
  • 开头两行的字典存储了join_types与end_types的类型。
  • 然后调用了clip里的一些东西。
  • S=ensure_list确保数据统一性，S=conv_to将数据倍乘变整数，确保能够被处理。最后调用一堆方法完成平移与交点处理，然后数据复原(除)返回。

clean_polygon函数

  • 首先还是conv_to数据转化，这里传入一个图形P的点集，取出来后调用方法清理，主要是冗余的数据(重复的点、类似三点一线可省略中间点的类型…)，最后还原倍数返回。
  • 为什么要清理呢？节省计算时间提高效率。

op函数

  • 这里是最关键的操作部分，称作多边形的布尔运算(交集、并集、差集)；A主体；B刀具；clip_type填充规则，这里说是假如图形的线段自己相交，哪里算外面，哪里算里面。
  • 通过双重保险(ensure_list和if…list)确保数据一定是列表。同时也是两个字典去放数据

收获

  1. 用户输入的数据不一定是能够直接使用，要考虑到不同的表现形式，最终决定是标注提示还是后台统一标准。
  2. DFS递归算法。
  3. 为了代码的简略或美观，不一定要用return传回数值。
  4. 数据整洁性处理(去噪简化)。
