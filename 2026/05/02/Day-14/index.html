dec.py函数

dec函数

这部分为dec.py的主函数，用于调用之前的函数进行循环，删掉不重要的点。

参数分析

P为一个小图块的原始点集；maxvalue为重要性阈值，这边的删除逻辑是进行多轮，每一轮中将最小的影响值记录下来(删除这个点对图形的影响)，如果大于这个阈值就停止，不再进行下一轮删点；除了这个去限制删点环节之外还有最小点数num兜底，至少要保留这个数目的坐标点；keep_end开关是否要保持曲线的起点和终点；closed告诉是不是闭合曲线；get_indices/get_flags决定是否要返回这个过程中的一些数据点；keep是不能动的点的集合；perimeter是类似弹簧系数的东西，将图形做一个缩放操作，从而和maxvalue比较。

if num==0:num=2如果没设置最少点数，默认为2，其实感觉这边用默认参数的函数会好一点。哦！它这里可能传入的是动态变量，比如一个方程，可能计算结果也是0，所有这里用if会更好。s=np.arry(P)将点集转换成np矩阵，提高计算的效率。为什么要用`I=list(range(s.shape[0]))记录原来每个点的工号呢，np不是会给下标？因为删除的时候下标都在变化，np索引是动态的。

delval=[]准备空列表，记录被删除的点的relevance分数(点的重要性)。如果没有传入perimeter参数就将所有点之间的距离加起来(周长)作为标尺，如果传了就用perimeter。if closed则将第一个点复制一份到末尾，原始点集也进行同样操作，因为计算重要性的时候需要前后两个邻居。同时用n=s.shape[0]记录总点数，不然一直在变，记录个标准。

if keep判断有没有点比较特殊，有的话则把特殊位置记为True，这里keep是所有数据点，不特殊的话在keep=[k>0 for k in keep]判断里就是False，如果没有keep，则都是False。

if keep_end保留终点就将其设置为True。然后np.zero(n)是创建n个0数据点，如果是(n,2)则是生成n行2列的零矩阵。这个用来记录每个点的重要性。然后value[i]=relevance(s,...)调用，获取数据。

其次进去核心删点环节，while True先套个无限循环，同时这个也是确保每一轮只删除一个点，因为不知道要删除多少个点，所以这样会更好。虽然可以判断，但要多写代码。内部如果说剩余点数小于预设if num>=n:则退出循环，i=np.argmin(value)得到重要性最低的点，和预设比较。bf=blocked(s,i)判断这个点能不能删除，如果blocked(阻碍)，就进入下面代码找备胎。ind=np.argsort(value)得到按分数从小到大排列时从小到大的数据对应的原始下标，这个类型是数组。rel=value[ind]是np里的花式索引，通过索引列表获取数据，所以rel现在是从小到大的排好序的数据列表。j=1从第二个下标开始找，因为第一个点通过blocked判断不能删。然后while j<n:这一部分是循环直到下一个通过blocked判断的点查找出来。

然后获取删除点的坐标数据x和ypx=s[i,0] py=s[i,1]，通过s=np.delte(s,i,0)和I=np.delete(I,i,0)将这个能够删除的点从原始点集和工号点集之中删掉，例如删掉s里第i个数据这一行(0)，每个点的数据占一行。同时在keep里弹出i这个数据的布尔值，在value重要性数据中也删掉这个东西。将这个点存在delval这个删除数据表之中。

之后i<sub>0</sub>=i-1和i<sub>1</sub>=i定位被删点的邻居，通过if i<sub>0</sub>==-1和elif i<sub>1</sub>=0这两个处理首尾越界问题。

然后value[i<sub>0</sub>]这个重新获取邻居的重要性(因为前后邻居变了)，另外一个邻居也是这么计算。

整个删点循环判断完之后，如果是闭合图形，裁掉原始点集P、计算矩阵s和工号列表I中增加的终点数据。

flags=[1,if i in I else 0 for i in range(P.shape[0])]是得到原始点集P里每个点是被删除还是留存下来了。if get_flags就是判断用户需不需要这个数据。if get_indices:如果用户需要简化后的坐标还有原来排第几就返回S和I。

疑惑

在他的这个删点循环判断过程中，如果说在判断这个点删除后会自交，然后找备胎的时候找到特别后面。那他把这个备胎删掉之后计算完备胎的邻居重要性还要再返回到之前不能够删除的这个点上。那假如说这个备胎删掉之后他还是会自交。还要再进行一大堆的判断。 我感觉会是比较冗余的。

然后他这里给了一个优化思路，就是在进行循环判断的过程中，如果说会自交，就给他一个标签——新建立一个列表数据来存储这个点它能不能够删除。他给的思路是说如果这个点他的邻居没有变化，他这个标签就一直不变，那这边会有一个疏漏就是。 决定他会不会自交的这个点，不仅仅是他的邻居。

然后我让他优化了，结果就是在判断能不能够删除的时候返回它会和哪条线段自交。那如果说动了这些线的点，再去进行状态的调整。

这边还有一个细节就不一定是所有的点它都需要进行blocked的判断。如果说在获取到这个点，它是重要性最小的点，这个时候才需要去进行blocked判断。 同时呢，在删掉点的之后，不仅需要对他的邻居进行blocked状态的重置(如果说不是safe的时候)。还要呢去把那个会和经过这个点的线段自交的点的blocked进行重测。

dec.py函数总结

在保持图形基本特征不变的前提下把不重要的点进行删除，实现图形的简化。

收获

  1. 带默认参数的函数的使用场景如果是if num==0:num=2这样的时候要慎用，因为调用的时候或许会是动态变量，要具体分析。
  2. if maxvalue==0.0则就不进行删点操作，为什么不在外面判断，等于零的时候就不调用不进入这一个逻辑呢？在大工程的程序设计中各个板块负责的代码不同，不能将判断是否进行本函数的逻辑给到外部去判断，或者说这样就很依赖于主函数逻辑的健全性，可以是可以，但对于这个部分设计者来讲，这一步算是兜底，将主函数的依赖性降低。算是代码的健壮性。
